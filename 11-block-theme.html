Thèmes par blocs

Full Site Editing (FSE)

L'éditeur de blocs

Depuis WordPress 5.0 (2018), l'éditeur Gutenberg est l'éditeur par défaut. Il permet de composer des pages et des articles à l'aide de blocs.

Depuis WordPress 5.8 (2021), le Full Site Editing (FSE) étend cet éditeur à l'ensemble du site : header, footer, templates de page...

Documentation Block Editor

Thème classique vs thème par blocs

Thème classique Thème par blocs
Templates Fichiers PHP Fichiers HTML
Configuration functions.php theme.json
Édition Éditeur de code Éditeur de site (admin)
Personnalisation PHP / CSS / JS Blocs + CSS

Structure d'un thème par blocs

Un thème par blocs nécessite au minimum les fichiers suivants :

your-theme-name/
├── parts/              → Parties réutilisables (header, footer...)
│   ├── header.html
│   └── footer.html
├── templates/          → Templates de page
│   └── index.html      → Template par défaut (obligatoire)
├── style.css           → Informations du thème (obligatoire)
└── theme.json          → Configuration globale du thème
Documentation Block Themes

Le fichier theme.json

Le fichier theme.json est le fichier de configuration central d'un thème par blocs. Il permet de définir les couleurs, les typographies et les espacements disponibles dans l'éditeur.

{
  "$schema": "https://schemas.wp.org/trunk/theme.json",
  "version": 3,
  "settings": {
    "color": {
      "palette": [
        { "name": "Primary", "slug": "primary", "color": "#0073aa" },
        { "name": "Secondary", "slug": "secondary", "color": "#23282d" }
      ]
    },
    "typography": {
      "fontSizes": [
        { "name": "Small", "slug": "small", "size": "0.875rem" },
        { "name": "Normal", "slug": "normal", "size": "1rem" }
      ]
    }
  }
}

Les templates

Les templates sont des fichiers HTML situés dans le dossier templates/. Ils sont composés de commentaires WordPress représentant des blocs :

<!-- wp:template-part {"slug":"header"} /-->

<!-- wp:group {"tagName":"main"} -->
<main class="wp-block-group">

    <!-- wp:post-title /-->
    <!-- wp:post-content /-->

</main>
<!-- /wp:group -->

<!-- wp:template-part {"slug":"footer"} /-->

Les parties réutilisables (header, footer...) sont définies dans le dossier parts/ et incluses via le bloc wp:template-part.

L'éditeur de site

L'éditeur de site (Apparence > Éditeur) permet de modifier visuellement les templates et les parties du thème sans écrire de code.

  • Modification des templates existants
  • Création de nouveaux templates
  • Gestion des parties (header, footer...)
  • Personnalisation des styles globaux
Les modifications faites dans l'éditeur sont stockées en base de données

Créer un thème par blocs

Le plugin Create Block Theme permet de générer un thème par blocs directement depuis l'interface d'administration de WordPress.

Pour démarrer un thème manuellement, il est également possible de s'appuyer sur le thème officiel Twenty Twenty-Four comme référence.

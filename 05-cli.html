Command Line Interface

Installation

L’interface en ligne de commande vous permettra de réaliser plusieurs tâches plus rapidement ou même de façon automatique.

curl -O https://raw.githubusercontent.com/wp-cli/builds/gh-pages/phar/wp-cli.phar
chmod +x wp-cli.phar
sudo mv wp-cli.phar /usr/local/bin/wp

Pour Windows, ouvrez le Git Bash en Administrateur puis :

curl -O https://raw.githubusercontent.com/wp-cli/builds/gh-pages/phar/wp-cli.phar
chmod +x wp-cli.phar
mv wp-cli.phar /usr/bin/wp
Liste des commandes

Téléchargement et installation de WordPress

wp core download --locale=fr_FR
wp config create --dbhost=localhost:3306 --dbname=mydb --dbuser=root --dbpass=root
wp db create
wp core install --url=localhost --title="Mon blog" --admin_user=admin --admin_password=admin --admin_email=admin@gmail.com

Regénérer les thumbnail

Si vous ajouter un nouveau format d'image dans un thème WordPress, il est nécessaire de regénérer les images déjà existantes.

Des extensions permettent d'effectuer ce traitement mais l'utilisation de la ligne de commande peut être plus simple et plus rapide :

wp media regenerate

Sauvegarder un site

Un backup complet d'un site WordPress comprend deux éléments : la base de données et les fichiers.

Sur le serveur distant :

wp db export backup.sql
tar -czf backup-files.tar.gz wp-content/

Copier les fichiers en local puis restaurer :

wp db import backup.sql
tar -xzf backup-files.tar.gz
Documentation wp db export

Adapter le site en local

Après l'import, les URLs pointent encore vers le serveur distant. La commande wp search-replace remplace toutes les occurrences dans la base de données, y compris dans les données sérialisées :

wp search-replace 'https://mon-site.com' 'http://localhost:8000'

Cette commande met également à jour les options siteurl et home. On peut les vérifier ou les modifier manuellement :

wp option get siteurl
wp option get home
wp option update siteurl 'http://localhost:8000'
wp option update home 'http://localhost:8000'

Enfin, vider le cache :

wp cache flush

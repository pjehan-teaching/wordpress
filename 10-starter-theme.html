Utiliser un Starter Theme

Introduction

Un starter theme offre généralement les avantages suivants :

  • Une structure de fichiers prédéfinie
  • Des fichiers indispensables déjà présent (index.php, functions.php...)

Et peut éventuellement également proposer :

  • Un bundler pour la compilation des assets (Vite, Webpack...)
  • Un moteur de template (Twig, Blade...)
  • Un gestion des dépendances front (NPM) et back (composer)
  • Un système de live reloading
  • Un linter pour la validation du code
  • ...

Sage

Sage est un starter theme développé par une équipe de développeurs. Il s'agit d'un projet Open Source.

Il utilise Laravel Blade comme moteur de template, Vite pour la compilation des assets et s'appuie sur Acorn pour intégrer des fonctionnalités Laravel dans WordPress.

Il existe également un boilerplate Bedrock développé par la même équipe.

roots.io/sage

Création d'un nouveau thème

Pour créer un nouveau thème avec Sage, nous pouvons exécuter la suite d'instructions suivante :

mkdir wordpress
cd wordpress
wp core download --locale=fr_FR
wp config create --dbhost=localhost:3306 --dbname=dbname --dbuser=root --dbpass=root --locale=fr_FR
wp db create
wp core install --url=localhost:8080 --title=Example --admin_user=admin --admin_password=admin --admin_email=admin@example.com
cd wp-content/themes
composer create-project roots/sage your-theme-name
cd your-theme-name
npm install
npm run build

Démarrer le serveur de développement

La configuration du thème se fait dans le fichier vite.config.js. Il faut notamment y renseigner l'URL du site WordPress local :

/** @type {import('vite').UserConfig} */
export default defineConfig({
    // ...
    server: {
        host: 'localhost',
    },
});
npm run dev

Puis dans l'administration du site, activer le thème nouvellement installé.

Documentation Vite / compilation des assets

Structure

themes/your-theme-name/   + → Root of your Sage based theme
├── app/                  + → Theme PHP
│   ├── Providers/        + → Service Providers
│   ├── View/             + → View Composers and Components
│   ├── filters.php       # → Theme filters
│   └── setup.php         # → Theme setup (add_action() functions)
├── config/               - → Theme configuration
├── public/               - → Built theme assets (never edit)
├── resources/            # → Theme assets and templates
│   ├── css/              + → Theme CSS
│   ├── fonts/            + → Theme fonts
│   ├── images/           + → Theme images
│   ├── js/               + → Theme JS
│   └── views/            + → Blade templates
│       ├── components/   + → Blade components
│       ├── layouts/      + → Base templates
│       └── partials/     + → Partial templates
├── vendor/               - → Composer packages (never edit)
├── node_modules/         - → Node.js packages (never edit)
├── composer.json         # → Composer dependencies
├── functions.php         - → Composer autoloader
├── index.php             - → Never manually edit
├── package.json          # → Node.js dependencies and scripts
├── screenshot.png        + → Theme screenshot for WP admin
├── style.css             + → Theme meta information
└── vite.config.js        # → Vite configuration

View Composers

Les View Composers permettent de passer des données à un template Blade. Ils sont définis dans le dossier app/View/Composers/ et doivent suivre la hiérarchie de WordPress :

namespace App\View\Composers;

use Roots\Acorn\View\Composer;

class Single extends Composer
{
    protected static $views = ['single'];

    public function with(): array
    {
        return [
            'title' => get_the_title(),
        ];
    }
}

La variable $title sera alors disponible dans le template Blade single.blade.php.
